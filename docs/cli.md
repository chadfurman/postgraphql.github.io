    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-command-line-interface" class="anchor" href="#command-line-interface" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Command Line Interface</h1>
<p>The easiest way to get up and running with PostGraphQL is to use the Command Line Interface.</p>
<p>Just install PostGraphQL globally with npm:</p>
<div class="highlight highlight-source-shell"><pre>npm install -g postgraphql</pre></div>
<p>…and you will have the <code>postgraphql</code> command ready to use!</p>
<p>The usage of the <code>postgraphql</code> binary is as follows. To pull up this documentation on your machine simply run <code>postgraphql --help</code>.</p>
<pre><code>
  Usage: postgraphql [options...]

  A GraphQL schema created by reflection over a PostgreSQL schema 🐘

  Options:

    -h, --help                       output usage information
    -V, --version                    output the version number
    -c, --connection &lt;string&gt;        the Postgres connection. if not provided it will be inferred from your environment, example: postgres://user:password@domain:port/db
    -s, --schema &lt;string&gt;            a Postgres schema to be introspected. Use commas to define multiple schemas
    -w, --watch                      watches the Postgres schema for changes and reruns introspection if a change was detected
    -n, --host &lt;string&gt;              the hostname to be used. Defaults to `localhost`
    -p, --port &lt;number&gt;              the port to be used. Defaults to 5000
    -m, --max-pool-size &lt;number&gt;     the maximum number of clients to keep in the Postgres pool. defaults to 10
    -r, --default-role &lt;string&gt;      the default Postgres role to use when a request is made. supercedes the role used to connect to the database
    -q, --graphql &lt;path&gt;             the route to mount the GraphQL server on. defaults to `/graphql`
    -i, --graphiql &lt;path&gt;            the route to mount the GraphiQL interface on. defaults to `/graphiql`
    -b, --disable-graphiql           disables the GraphiQL interface. overrides the GraphiQL route option
    -t, --token &lt;identifier&gt;         the Postgres identifier for a composite type that will be used to create tokens
    -o, --cors                       enable generous CORS settings. this is disabled by default, if possible use a proxy instead
    -a, --classic-ids                use classic global id field name. required to support Relay 1
    -j, --dynamic-json               enable dynamic JSON in GraphQL inputs and outputs. uses stringified JSON by default
    -M, --disable-default-mutations  disable default mutations, mutation will only be possible through Postgres functions
    -l, --body-size-limit &lt;string&gt;   set the maximum size of JSON bodies that can be parsed (default 100kB) The size can be given as a human-readable string, such as '200kB' or '5MB' (case insensitive).
    --secret &lt;string&gt;                DEPRECATED: Use jwt-secret instead
    -e, --jwt-secret &lt;string&gt;        the secret to be used when creating and verifying JWTs. if none is provided auth will be disabled
    -A, --jwt-audience &lt;string&gt;      a comma separated list of audiences your jwt token can contain. If no audience is given the audience defaults to `postgraphql`
    --jwt-role &lt;string&gt;              a comma seperated list of strings that create a path in the jwt from which to extract the postgres role. if none is provided it will use the key `role` on the root of the jwt.
    --export-schema-json [path]      enables exporting the detected schema, in JSON format, to the given location. The directories must exist already, if the file exists it will be overwritten.
    --export-schema-graphql [path]   enables exporting the detected schema, in GraphQL schema format, to the given location. The directories must exist already, if the file exists it will be overwritten.
    --show-error-stack [setting]     show JavaScript error stacks in the GraphQL result errors

  Get Started:

    $ postgraphql --demo
    $ postgraphql --schema my_schema

</code></pre>
